# Changelog

All notable changes to `clain` are recorded here. Format follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/). The project is pre-1.0; semantic versioning will start applying once we cut a 0.1.0 release.

Entries cite the spec ID and title. Dates live in git history; author attribution lives in `pyproject.toml`.

## [Unreleased]

- spec 0017 ‚Äî *Tokyo Night theme (dark + light) with terminal-background detection*. New module `src/clain/ui/theme.py` defines a `Theme` dataclass of named tokens (`brand`, `brand_step1..5`, `safe`, `unsafe`, `warning`, `fix`, `fg`, `dim`, `accent`, `class_cache_managed`, `class_bytecode`, `class_ephemeral`, `class_unknown`) plus two instances (`TOKYO_NIGHT_DARK`, `TOKYO_NIGHT_LIGHT`) lifted from the upstream tokyonight palette. `resolve_theme(flag, env, colorfgbg, no_color, *, osc11)` implements the precedence ladder: `NO_COLOR` ‚Üí `--theme dark|light` ‚Üí `CLAIN_THEME` ‚Üí `COLORFGBG` detection ‚Üí OSC 11 query (best-effort, 50 ms timeout; opt-in) ‚Üí fallback dark. Unknown values raise `InvalidThemeValue`, surfaced by the CLI as a `user_error` naming the valid vocabulary. Every Rich-markup colour name in `src/clain/ui/tables.py` and `src/clain/ui/errors.py` is replaced by a theme token; a new `test_no_raw_color_names_in_renderers` (AST-based, ignores docstrings) pins the migration. `user_error` (spec 0015) now renders headlines in `theme.unsafe` and the fix line in `theme.fix`. No JSON change, no behaviour change in plan / executor / phase gate. New `--theme` global flag on `clain`.
- spec 0015 ‚Äî *Global installability + error-message sweep*. The supported end-user install path is now `pipx install git+‚Ä¶` (or `pixi global install --git ‚Ä¶`) ‚Äî both produce a `clain` binary on `PATH`. README, `docs/USAGE.md`, and `AGENTS.md` lead with the bare `clain` invocation; `pixi run clain ‚Ä¶` is confined to explicitly marked `<!-- contributor-only -->` sections (enforced by a sentinel-based test). The console-script entry point already shipped (`[project.scripts] clain = "clain.cli:app"`); this spec documents it as the headline path and pins it with `test_console_script_entry_resolves`. Error sweep: new `clain.ui.errors.user_error(what, why, fix)` helper codifies the spec-0013 `CLAIN_SYNCED_ROOT` template shape, and every `err_console.print` call site in `cli.py` either uses the helper or carries an `# error template OK` comment. Specific user-facing rewrites: `CLAIN_DEV_ROOT` unset now shows the `export` line and the `--here` alternative; non-existent / non-directory ROOT errors suggest `--here`; missing classify cache names the literal `clain classify <path>` fix; `plan explain` action-not-found suggests `jq '.actions[].id'`. No JSON schema change, no new CLI flags, no behaviour change in plan / executor / phase gate.
- spec 0008 ‚Äî *GitHub repository presentation + discoverability*
- spec 0009 ‚Äî *Rule base completeness*: `.pixi` cache-managed, bare `venv` dropped, `.git`-family pruned, nullable `in_sync_tree`. Schema-1 additive: `in_sync_tree` may now be `null` when `CLAIN_SYNCED_ROOT` is unset.
- spec 0010 ‚Äî *Single-workspace mode*: `--here` flag on `classify` / `plan recreate` / `plan move`. Rich `Tree` renderer for single-workspace classify. `scan.mode = "tree" | "single"` (additive; absent = `"tree"` for backwards compat).
- spec 0011 ‚Äî *Documentation generalisation*: synced-storage framing (GDrive / OneDrive / Dropbox / iCloud Drive as the class, not just GDrive), single-workspace-first quickstart, captured Rich output, reproducible via `examples/capture.py`.
- spec 0012 ‚Äî *Plan presentation*: workspace-grouped Rich Panels with relative `Target` / `Command(s)` (default render), and `--table` to keep the pre-0012 single-table layout for copy-paste / spreadsheet use. `--table` and `--json` are mutually exclusive. Persisted plan JSON is unchanged (absolute paths preserved); relativisation is render-only.
- spec 0014 ‚Äî *Vertical rhythm + 0013 follow-through fixes*. **Part A (correctness fixes from 0013 fallout):** classify cache filenames gain a `-v<schema>` suffix and the classify schema bumps to **2** so stale caches from before spec 0013 invalidate cleanly ‚Äî old `<root-hash>.json` / `<root-hash>-v1.json` files are removed on access. Four remaining `CLAIN_SYNCED_ROOT` references in `src/clain/ui/tables.py` user-visible strings are swept (the startup gate already rejected the env var, but render hints still mentioned it). The legacy `classify_footer` / `single_workspace_tree` / `single_workspace_footer` renderers (unused since 0013) are removed. **Part B (vertical rhythm):** meta lines `(cached ‚Ä¶)` / `(dry mode ‚Ä¶)` render with one blank line above and indent to `BODY_INDENT` instead of column 0; horizontal rules become fixed-measure (`RULE_WIDTH=72`) with one blank line above and below; class headers on `classify --here` use a hanging-indent form (count on its own line, description and members aligned below); the classify Key block converges to the multi-line form already used by plan view; every render ends with a trailing blank line. New `src/clain/ui/rhythm.py` exposes the four typography constants (`SECTION_GAP` / `META_GAP` / `BODY_INDENT` / `RULE_WIDTH`). No JSON schema break, no new CLI flags, no behaviour change in plan / executor / phase gate. Cache files regenerate on next scan; no user action needed.
- spec 0013 ‚Äî *Output legibility + sync-placement autodetect*: orientation headers, inline legends with `--legend` / `--no-legend` / `CLAIN_LEGEND` (default *on* for `--here`, *off* for tree mode), more breathing room around panels and classify renders, macOS sync-placement autodetection against six known synced-storage path patterns. **Removed `CLAIN_SYNCED_ROOT`** ‚Äî it conflated workspace-parent and sync-claim into one env var, and autodetect covers the real cases. If still set in the environment, the CLI hard-errors with a pointer to unset it. The classify JSON gains a `sync_placement` block (`state` / `provider` / `source` / `synced_root`); the legacy `in_sync_tree` boolean stays and aligns with `sync_placement.state`.
