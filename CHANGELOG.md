# Changelog

All notable changes to `clain` are recorded here. Format follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/). The project is pre-1.0; semantic versioning will start applying once we cut a 0.1.0 release.

Entries cite the spec ID and title. Dates live in git history; author attribution lives in `pyproject.toml`.

## [Unreleased]

- spec 0016 — *Orientation identity (brand meter + command emoji + intent prose) + `type → action` rename*. **Part A — orientation.** Every primary render now opens with a five-block brand meter, the `clain` wordmark, a per-command emoji, and the command name + flags, followed by a plain-English intent line describing what the command *does*. The meter encodes workflow position (`classify`=2/5, `plan … --dry`=3/5, `plan explain`=4/5, execute=5/5) and the five blocks pick up the `theme.brand_step1..5` gradient from spec 0017 (cyan → red). The legacy `clain classify --here  →  one-workspace classification` header is gone. Identity metadata (level, emoji, name, intent) lives in `src/clain/ui/intent.py` as a `CommandIdentity` mapping keyed by stable strings (`classify_here`, `plan_recreate_dry`, …); meter / anchor / banner renderers live in `src/clain/ui/banner.py`. **First-run banner.** `classify` (single or tree) shows a full-width ASCII `clain` banner with tagline + repo URL on the first invocation per machine, gated by a marker file at `$XDG_STATE_HOME/clain/banner-shown`. New `--banner` / `--no-banner` flags on `classify` (mutually exclusive) and a `CLAIN_BANNER=on|off` env var; `--json` mode always suppresses the banner. `--banner` (force-show, e.g. for screenshots) deliberately does NOT consume the marker. **Part B — `type → action` rename + schema bump.** Plan JSON now uses `"action"` instead of `"type"` for the action-category field; `SCHEMA_VERSION` bumps to **2**; persisted plan filenames adopt the spec-0014 schema-versioned pattern (`<kind>-<UTC>-v<schema>.json`). `plan explain` refuses plan files whose `schema` doesn't match and prints a regenerate prompt. New `prune_stale_plan_files(current_schema)` in `state.py` removes plan files older than 7 days whose schema is stale, called on every plan save. The renderer column header is now `Action`; `tests/snapshots/plan_table_flat.{fixture.json,txt}` are updated in lockstep. No behaviour change in plan / executor / phase gate. No new CLI flags on `plan` / `plan recreate` / `plan move` / `plan explain`.
- spec 0017 — *Tokyo Night theme (dark + light) with terminal-background detection*. New module `src/clain/ui/theme.py` defines a `Theme` dataclass of named tokens (`brand`, `brand_step1..5`, `safe`, `unsafe`, `warning`, `fix`, `fg`, `dim`, `accent`, `class_cache_managed`, `class_bytecode`, `class_ephemeral`, `class_unknown`) plus two instances (`TOKYO_NIGHT_DARK`, `TOKYO_NIGHT_LIGHT`) lifted from the upstream tokyonight palette. `resolve_theme(flag, env, colorfgbg, no_color, *, osc11)` implements the precedence ladder: `NO_COLOR` → `--theme dark|light` → `CLAIN_THEME` → `COLORFGBG` detection → OSC 11 query (best-effort, 50 ms timeout; opt-in) → fallback dark. Unknown values raise `InvalidThemeValue`, surfaced by the CLI as a `user_error` naming the valid vocabulary. Every Rich-markup colour name in `src/clain/ui/tables.py` and `src/clain/ui/errors.py` is replaced by a theme token; a new `test_no_raw_color_names_in_renderers` (AST-based, ignores docstrings) pins the migration. `user_error` (spec 0015) now renders headlines in `theme.unsafe` and the fix line in `theme.fix`. No JSON change, no behaviour change in plan / executor / phase gate. New `--theme` global flag on `clain`.
- spec 0015 — *Global installability + error-message sweep*. The supported end-user install path is now `pipx install git+…` (or `pixi global install --git …`) — both produce a `clain` binary on `PATH`. README, `docs/USAGE.md`, and `AGENTS.md` lead with the bare `clain` invocation; `pixi run clain …` is confined to explicitly marked `<!-- contributor-only -->` sections (enforced by a sentinel-based test). The console-script entry point already shipped (`[project.scripts] clain = "clain.cli:app"`); this spec documents it as the headline path and pins it with `test_console_script_entry_resolves`. Error sweep: new `clain.ui.errors.user_error(what, why, fix)` helper codifies the spec-0013 `CLAIN_SYNCED_ROOT` template shape, and every `err_console.print` call site in `cli.py` either uses the helper or carries an `# error template OK` comment. Specific user-facing rewrites: `CLAIN_DEV_ROOT` unset now shows the `export` line and the `--here` alternative; non-existent / non-directory ROOT errors suggest `--here`; missing classify cache names the literal `clain classify <path>` fix; `plan explain` action-not-found suggests `jq '.actions[].id'`. No JSON schema change, no new CLI flags, no behaviour change in plan / executor / phase gate.
- spec 0008 — *GitHub repository presentation + discoverability*
- spec 0009 — *Rule base completeness*: `.pixi` cache-managed, bare `venv` dropped, `.git`-family pruned, nullable `in_sync_tree`. Schema-1 additive: `in_sync_tree` may now be `null` when `CLAIN_SYNCED_ROOT` is unset.
- spec 0010 — *Single-workspace mode*: `--here` flag on `classify` / `plan recreate` / `plan move`. Rich `Tree` renderer for single-workspace classify. `scan.mode = "tree" | "single"` (additive; absent = `"tree"` for backwards compat).
- spec 0011 — *Documentation generalisation*: synced-storage framing (GDrive / OneDrive / Dropbox / iCloud Drive as the class, not just GDrive), single-workspace-first quickstart, captured Rich output, reproducible via `examples/capture.py`.
- spec 0012 — *Plan presentation*: workspace-grouped Rich Panels with relative `Target` / `Command(s)` (default render), and `--table` to keep the pre-0012 single-table layout for copy-paste / spreadsheet use. `--table` and `--json` are mutually exclusive. Persisted plan JSON is unchanged (absolute paths preserved); relativisation is render-only.
- spec 0014 — *Vertical rhythm + 0013 follow-through fixes*. **Part A (correctness fixes from 0013 fallout):** classify cache filenames gain a `-v<schema>` suffix and the classify schema bumps to **2** so stale caches from before spec 0013 invalidate cleanly — old `<root-hash>.json` / `<root-hash>-v1.json` files are removed on access. Four remaining `CLAIN_SYNCED_ROOT` references in `src/clain/ui/tables.py` user-visible strings are swept (the startup gate already rejected the env var, but render hints still mentioned it). The legacy `classify_footer` / `single_workspace_tree` / `single_workspace_footer` renderers (unused since 0013) are removed. **Part B (vertical rhythm):** meta lines `(cached …)` / `(dry mode …)` render with one blank line above and indent to `BODY_INDENT` instead of column 0; horizontal rules become fixed-measure (`RULE_WIDTH=72`) with one blank line above and below; class headers on `classify --here` use a hanging-indent form (count on its own line, description and members aligned below); the classify Key block converges to the multi-line form already used by plan view; every render ends with a trailing blank line. New `src/clain/ui/rhythm.py` exposes the four typography constants (`SECTION_GAP` / `META_GAP` / `BODY_INDENT` / `RULE_WIDTH`). No JSON schema break, no new CLI flags, no behaviour change in plan / executor / phase gate. Cache files regenerate on next scan; no user action needed.
- spec 0013 — *Output legibility + sync-placement autodetect*: orientation headers, inline legends with `--legend` / `--no-legend` / `CLAIN_LEGEND` (default *on* for `--here`, *off* for tree mode), more breathing room around panels and classify renders, macOS sync-placement autodetection against six known synced-storage path patterns. **Removed `CLAIN_SYNCED_ROOT`** — it conflated workspace-parent and sync-claim into one env var, and autodetect covers the real cases. If still set in the environment, the CLI hard-errors with a pointer to unset it. The classify JSON gains a `sync_placement` block (`state` / `provider` / `source` / `synced_root`); the legacy `in_sync_tree` boolean stays and aligns with `sync_placement.state`.
